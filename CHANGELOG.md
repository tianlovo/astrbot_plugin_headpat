# Changelog

## v1.3.0 (2026-03-22)

### 新增
- **GIF 缓存服务**: 新增独立的 `service/` 包结构，提供 GIF 缓存管理功能
  - 缓存存储：基于用户ID的GIF缓存存储，遵守 AstrBot 大文件存储规范
  - 缓存读取：优先返回缓存的GIF，提高响应速度
  - 缓存更新：支持刷新和替换缓存
  - 过期清理：基于 TTL 的自动过期清理（可配置）
  - LRU 清理：容量达到上限时自动清理最久未使用的缓存
  - 自动清理：支持定时自动清理过期缓存（可配置）
  - 启动清理：支持启动时执行一次过期缓存清理（可配置）
- **缓存配置项**:
  - `cache_enabled`: 是否启用GIF缓存
  - `cache_ttl`: 缓存有效期（秒），默认3600秒
  - `max_cache_size`: 最大缓存文件数量，默认100个
  - `auto_cleanup`: 是否启用自动过期清理
  - `cleanup_interval`: 自动清理间隔（秒）
  - `cleanup_on_startup`: 启动时是否执行一次过期清理

### 架构
- **模块化设计**: 缓存服务与业务逻辑解耦，通过清晰接口交互
- **高复用性**: 缓存模块可被项目中多个组件调用
- **完整文档**: 提供模块使用说明、接口定义和注意事项
- **单元测试**: 添加完整测试覆盖缓存功能

## v1.2.1 (2026-03-22)

### 新增
- **透明背景配置**: 新增 `transparent_background` 配置项，默认启用透明背景
- **背景颜色配置**: 新增 `background_color` 配置项，当不启用透明背景时可指定背景颜色（默认白色 #FFFFFF）
- **颜色解析**: 支持十六进制颜色值，包括短格式（#RGB）和标准格式（#RRGGBB）

## v1.2.0 (2026-03-22)

### 变更
- **作者更新**: 从 `mjy1113451` 改为 `tianluoqaq`
- **仓库地址更新**: 从 `https://github.com/mjy1113451/touch_head` 改为 `https://github.com/tianlovo/astrbot_plugin_headpat`
- **插件名称**: 从 `astrbot_plugin_touch_head` 改为 `astrbot_plugin_headpat`

### 新增
- **QQ群白名单配置**: 新增 `allowed_groups` 配置项，只有在白名单列表中的群才能触发插件功能，为空则允许所有群

### 修改
- **触发规则重构**:
  - 移除 `.petset` 命令（配置改为通过 WebUI 管理）
  - 移除拍一拍事件监听
  - 移除普通消息前缀匹配触发
  - 改为使用 AstrBot 标准命令系统: `@filter.command("摸头", alias={"摸摸", "摸", "摸头杀"})`
  - 需要 at 机器人后才能触发命令
  - 支持命令别名: 摸摸、摸、摸头杀
  - 匹配以命令或别名开头的文本，如 "摸摸可以吗"

- **目标用户选择逻辑**:
  - 如果消息中只有一个 at（机器人自己），生成机器人自己的摸头 GIF
  - 如果消息中有多个 at，生成第二个 at 用户（非机器人）的摸头 GIF

### 移除
- `.petset` 命令（速度、触发词设置改为 WebUI 配置）
- 拍一拍事件监听功能
